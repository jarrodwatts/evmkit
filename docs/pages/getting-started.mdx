## Terminal Setup

The recommended way to use this project is to open **three** terminals simultaneously:

1. Runs the local node to deploy your smart contracts to
2. Is free to use for things like deploying smart contracts, running tests, git, yarn, etc.
3. Runs the frontend application

### Terminal One: Local Node

Terminal one essentially mimics the behavior of an EVM blockchain on your
local machine using [Hardhat](https://hardhat.org/).

> Hardhat comes built-in with Hardhat Network, a local Ethereum network node designed for development.
> It allows you to deploy your contracts, run your tests and debug your code,
> all within the confines of your local machine.

To set this, up run the following commands:

```bash
cd contracts      # Change directories into the contracts folder
yarn              # Install dependencies (only required the first time you set up the project)
yarn run-node     # Start the local node
```

### Terminal Two: Miscellaneous

The second terminal is free to use for things like deploying smart contracts, running tests, git, yarn, etc.

To set up the project, we&rsquo;ll use it to deploy the smart contracts to the local node.

Run the following commands:

```bash
cd contracts          # Change directories into the contracts folder
yarn deploy-local     # Deploy the smart contracts to the local node
```

This command uses a [Hardhat script](https://hardhat.org/guides/scripts.html)
to deploy the smart contracts to the local node, and update the frontend application
with the addresses and ABIs of the deployed contracts automatically.

Each time you make a change to your smart contracts,
you can run this command to redeploy them to the local node and
update the frontend application.

Some other useful commands you can run in this terminal:

```bash
yarn build           # Compile the smart contracts and detect what features are implemented
yarn test            # Run the smart contract test suite
```

### Terminal Three: Frontend

Terminal three runs the frontend of your application using [Next.js's CLI](https://nextjs.org/docs/api-reference/cli).

Run the following commands:

```bash
cd application          # Change directories into the application folder
yarn                    # Install dependencies (only required the first time you set up the project)
yarn dev                # Start the frontend application
```

Visit http://localhost:3000 in your browser to see the application running.

> `next dev` starts the application in development mode with hot-code reloading, error reporting, and more:
> The application will start at http://localhost:3000 by default. The default port can be changed with -p

Leave this terminal running in the background.

## Add a burner private key

<Callout type="info">
  This step is only required if you want to use Sign in with Ethereum.
</Callout>

To authenticate users using Sign in with Ethereum, you need a private key in your environment.

**Use a burner wallet. NOT your main wallet. The account does not need any funds.**

1. Create a file called `.env.local` at the root of the `application` folder.
2. Populate it as follows _(replace `xyz` with your PK)_: `ADMIN_PRIVATE_KEY=xyz`
